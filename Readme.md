# Aegis Student Guard

[![AMD Slingshot Hackathon](https://img.shields.io/badge/Hackathon-AMD%20Slingshot-black?style=for-the-badge&logo=amd)](https://github.com/nillohitroy/AMD-Slingshot)

Aegis Student Guard transforms campus security by moving beyond passive training to real-time active defense. Our browser extension acts as a decentralized sentinel, using local inference to intercept threats at the point of click. Instead of simple blocks, we generate instant educational explanations that turn attacks into teachable moments. By quantifying behavior into a dynamic risk score and automating phishing drills, Aegis evolves the student body from a vulnerable target into a hardened human firewall. 

Developed by **Team Challengers** for the AMD Slingshot Hackathon, this project leverages the power of Next.js, Tailwind CSS, Django, and AMD hardware optimizations to deliver a scalable, cognitive cybersecurity ecosystem.

---

## 🚀 Key Features

* **Real-Time Active Protection:** A lightweight Chrome extension that intercepts phishing and malware locally with near-zero latency.
* **Teachable Moments:** Replaces dead-end "Blocked" pages with interactive, AI-generated explanations of the threat.
* **Dynamic Risk Scoring:** Calculates a real-time behavioral resilience score (0–100) for every student.
* **Autonomous AI Phishing Drills:** Generates hyper-personalized, department-specific phishing simulations to train students against modern threats.
* **Dark Web Credential Monitoring:** Proactively scans global data breaches for compromised institutional emails.

---

## 🛠️ Technology Stack

* **Frontend:** Next.js (App Router), Tailwind CSS, Framer Motion, Recharts
* **Backend:** Python, Django 5.0, Django REST Framework
* **Extension:** Chrome Extension Manifest V3, JavaScript
* **AI & Intelligence:** Google Gemini Pro API (Cognitive Engine)
* **Hardware Optimization (Target):** AMD Ryzen™ AI (Client Edge), AMD EPYC™ with SEV-SNP (Confidential Backend Computing)

## ⚙️ Local Setup & Installation

Follow this straightforward guide to get the complete Aegis ecosystem running locally.

### Prerequisites
* Python 3.12
* Node.js 
* Google Chrome or Chromium-based browser

### 1. Backend Setup (Django)
The intelligence core that handles the APIs, AI generation, and database interactions.

```bash
# Navigate to the backend directory
cd backend

# Create and activate a virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Run database migrations
python manage.py makemigrations
python manage.py migrate

# Start the Django development server
python manage.py runserver
```
*The backend will be live at `http://localhost:8000`*

*Create a .env file in the backend folder, then create a key called GEMINI_API_KEY and insert your Gemini Api key there*

### 2. Frontend Setup (Next.js)
The client-facing applications built simply with modern Next.js and Tailwind CSS standards.

```bash
# Open a new terminal and navigate to the frontend directory
cd frontend
cd aegis

# Install dependencies
npm install

# Start the Next.js development server
npm run dev
```
*The frontend will be live at `http://localhost:3000`*

### 3. Chrome Extension Installation (The Sentry)
The decentralized edge sensor that monitors student web traffic.

1. Open Google Chrome and navigate to `chrome://extensions/`.
2. Enable **Developer mode** using the toggle in the top right corner.
3. Click the **Load unpacked** button in the top left.
4. Select the `aegis-extension` folder located in the project repository.
5. Pin the Aegis Sentry extension to your browser toolbar.

## 🖥️ Platform Usage Guide

The Aegis ecosystem operates through three distinct dashboards, serving different roles within the institution.

### 1. The Student Dashboard
**Access:** Navigate to `http://localhost:3000/student` (or your configured student route).

This is the primary interface for the end-user. 
* **Risk Score & Streak:** Monitor daily security hygiene and maintain a protection streak.
* **Recent Activity:** View a history of intercepted threats and completed drills.
* **Teachable Moments:** Review the AI reasoning behind any site that the Sentry extension recently blocked.

### 2. The Institution Admin Dashboard
**Access:** Navigate to `http://localhost:3000/admin`.

The command center for university IT and security teams.
* **Departmental Heatmaps:** View aggregated risk scores across different departments (e.g., Engineering vs. Arts) to identify vulnerabilities.
* **Cognitive Logs:** Read real-time, natural-language explanations generated by the AI for every threat encountered on campus.
* **AI Drill Generator:** Launch targeted, AI-crafted simulated phishing campaigns to at-risk student groups.

### 3. God Mode (Platform Administrator)
**Access:** Requires setting up a superuser account.

This dashboard provides top-level oversight, allowing you to manage global health, tenant configurations, and raw database entries.

**To enable God Mode:**
```bash
# In your backend terminal (with the virtual environment activated)
python manage.py createsuperuser

# Follow the prompts to create your admin email and password
```
Once created, log in to the Django Admin panel at `http://localhost:8000/admin` to directly manipulate user data, adjust policy configurations (`PolicyConfig`), review full audit logs, and trigger manual dark web scans.

## 🛡️ AMD Hardware Integration

Aegis is architected to utilize the next generation of AMD hardware:
* **Ryzen™ AI NPUs:** Shifts the extension's threat detection inference from the cloud to the local device edge, ensuring student browsing data remains private while achieving near-zero latency.
* **AMD EPYC™ Processors:** Our production backend targets instances equipped with SEV-SNP (Secure Encrypted Virtualization) to provide confidential computing, ensuring sensitive student risk data and cognitive logs are encrypted even while in memory.
